1) Упражнение 2.24. Какие из следующих определений (если они есть) недопустимы? Почему? Как их исправить?
(a)	int ival = 1.01;
(b) int &rvall = 1.01;
(с)	int &rval2 = ival;
(d) const int &rval3 = 1;

Ответ: Недопустимым является b, так как не константные ссылки нельзя инициализировать выражением rvall
       Исправить можно, если ссылку rvall объявить константой, const int &rvall


2) Упражнение 2.25. Какие из следующих присвоений (если они есть) являются недопустимыми 
(с учетом предыдущих определений)? Если таковые есть, объясните, почему.
(а)	rva12 = 3.14159;
(b) rva12 = rva13;
(с)	ival = rval3;
(d) rval3 = ival;

Ответ: Недопустимым язвляется b, так как мы не можем изменить константный указатель



3) Упражнение 2.26. В чем разница между определениями (а) и присвоениями (Ь)? 
Какие из них (если есть) недопустимы?
(a)	int ival = 0;	(b) ival = ri;
const int &ri =0;	ri = ival;

Ответ: Присвоение - это запись в выделенную память какого-то значения
       Определения - выделение памяти для хранения переменной какого-то типа
       Недопустимым является a, потому чтo у ссылки нет нулевого значения 



4) Упражнение 2.27. Что отобразит на экране следующий код?
int i, &ri = i; i = 5; ri = 10;
std::cout << i << " " << ri << std::endl;

Ответ: на экране мы получим,  10 10 

[1] Упражнения раздела 4.2.3
5) Упражнение 4.15. Укажите принципиальные различия между указателями и ссылками.

Ответ: 1. Нельзя объявить массив ссылок.
       2. У ссылки нет адреса.
       3. Существует арифметика указателей, для ссылок нет арифметики .
       4. Ссылка, в отличии от указателя, не может быть неинициализированной.
       5. Ссылка не может быть изменена после инициализации.
       6. У ссылки нет нулевого значения

6) Упражнение 7.7. Объясните различие между параметрами в следующих двух объявлениях.
void f(Т);
void f(Т&);

Ответ: 1 параметр является локальным, мы предаем значение переменной 
       2 параметр является глобальным, то есть мы передаем ссылку, которая может менять значение параметра

Приведите пример случая, когда параметр не должен быть ссылочным.

8) Упражнение 7.10. Следующая функция хоть и вполне допустима, но менее полезна,
чем могла бы быть. Выявите и устраните ограничения.
bool test(string& s) { return s.empty(); } Написать main для аргументации своего решения



9) Упражнение 7.11. Когда ссылочные параметры должны быть константными? 
Какие проблемы могут возникнуть, если параметр является обычной ссылкой, 
а какие — если он является константной ссылкой?

Ответ: Если параметр является обычной ссылкой, они связанны с объектом  такого же типа. 
       Если параметр является константной ссылко, мы не сможем его изменить
